install · getting started

From zero to first scan in 5 minutes.

One pip install, one command, one self-contained HTML report. No account. No cloud. No telemetry. Your code never leaves your machine.

cli-reference.txtphase 0 · mvp
commandpurposephase
project-mri analyze <path> Run a full MRI scan on a local repo. Outputs CLI summary + HTML report. MVP
project-mri dashboard Open the interactive Tauri dashboard for a previously-scanned repo. P2
project-mri mcp serve Start an MCP server so AI agents can query MRI live. P3
project-mri report <repo> --format md Export the analysis as Markdown (for wikis, PR descriptions). MVP
project-mri report <repo> --format json Export as machine-readable JSON (for scripts, dashboards). MVP
project-mri cache clear <repo> Wipe the local cache. Forces a full re-scan on next run. MVP
project-mri config Show / edit the current config (language filters, history depth, etc.). MVP
project-mri plugins list List installed community analyzer/scoring plugins. P3
/// configuration

Configure MRI to match your workflow.

~/.config/project-mri/config.tomltoml
# project-mri configuration · v0.1

[general]
log_level = "info"           # debug | info | warn | error
cache_dir = "~/.cache/mri"   # default
parallel_jobs = 4           # set to 0 for auto-detect

[git]
history_depth = "all"        # or "5y", "1000 commits", etc.
include_merges = true
churn_windows = ["30d", "90d", "365d"]

[code]
languages = ["python", "typescript", "go"]
exclude_paths = [
  "node_modules",
  "venv",
  "dist",
  "build",
  "*.test.ts",
]
complexity_threshold = 15     # cyclomatic

[scoring]
weights = {            # custom score weights (optional)
  architecture_health  = 0.25,
  technical_debt       = 0.30,
  bus_factor           = 0.20,
  coupling_evolution   = 0.25,
}

[ai_impact]
mode = "heuristic"        # heuristic | session_logs | off
session_log_paths = []
local_llm = "ollama:qwen2.5-coder:7b"  # optional

[output]
html = { template = "default", embed_assets = true }
cli = { color = true, progress = true }
/// next steps

Once you've run your first scan.

  1. Read the report critically. MRI explains why each score is what it is. Click into signals. Don't trust the number — understand the data.
  2. Run it on day 1, then again on day 30. The first scan is a snapshot. The trend is what matters.
  3. Wire it into CI. Block merges when architecture_health drops below your threshold. Use the JSON output for custom checks.
  4. Invite your team to share the report. A self-contained HTML file works as a Slack attachment or wiki page.
  5. Track AI influence over time. If ai_influence jumps from 15% to 45% in a quarter, that's a conversation worth having.
