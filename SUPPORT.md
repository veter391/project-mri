# Getting support

Thanks for using MRI. This document explains where to get help so your
question reaches the right place and gets answered efficiently.

MRI is maintained by a single person in their spare time. Please be patient
and respectful. Choosing the correct channel below is the fastest way to get
help and keeps the issue tracker focused on confirmed bugs.

## Before you ask

1. Read the [README](README.md) and the documentation it links to.
2. Search [existing issues](../../issues?q=is%3Aissue) and
   [Discussions](../../discussions) — your question may already be answered.
3. Make sure you are on the latest released version.

## Where to go

| I want to...                                   | Go to |
|------------------------------------------------|-------|
| Ask a question or get usage help               | [Discussions → Q&A](../../discussions/categories/q-a) |
| Share an idea or early feedback                | [Discussions → Ideas](../../discussions/categories/ideas) |
| Report a reproducible bug                      | [Open a bug report](../../issues/new?template=bug_report.yml) |
| Request a feature                              | [Open a feature request](../../issues/new?template=feature_request.yml) |
| Report a security vulnerability                | **Privately** — see [SECURITY.md](SECURITY.md). Never open a public issue. |
| Contribute code or docs                        | [CONTRIBUTING.md](CONTRIBUTING.md) |

## What to include when asking for help

The more of this you provide, the faster you will get a useful answer:

- MRI version (`mri --version`) and how you installed it.
- Operating system and Python version.
- The exact command you ran and the full output (including any error).
- What you expected to happen instead.
- A minimal example that reproduces the problem, if possible.

## Service expectations

This is a community, best-effort project. There is no paid support tier and no
guaranteed response time. Well-scoped, reproducible reports are prioritized.
