about · mission · community

An MRI scan for every codebase.

Project MRI exists because every engineer eventually inherits a codebase with no context. Every team eventually loses the "why" behind a decision. We think the tooling for understanding should be local, free, and open. Forever.

/// mission

What we're building toward.

Modern codebases are living things — they grow, decay, accumulate complexity, and quietly accumulate knowledge in single people's heads. Most engineering teams have no way to see this happening. They discover it the hard way: a critical contributor leaves, a feature that "just worked" suddenly breaks, a refactor that should have been easy turns out to be impossible.

The existing tools give you fragments — a Sonar dashboard for code style, a CodeScene chart for churn, a github blame for ownership. None of them tell the full story. None of them let you say: "show me the risk in this codebase, explained".

MRI combines those fragments into a single explainable model. It runs on your machine, in your time, without uploading anything. It produces a report that survives the tool itself — a file you can archive, share, and re-read in 5 years.

/// philosophy

Three convictions.

01
Software belongs to its authors.

Not to a vendor. Not to a platform. Not to the LLM-of-the-month. Tools that analyze your code should respect that ownership — by default.

02
Truth is more useful than flattery.

An AI that always agrees with you isn't helpful. A scoring system that hides its reasoning isn't trustworthy. MRI shows the data — you make the call.

03
Infrastructure outlives products.

Git is infrastructure. grep is infrastructure. Make is infrastructure. MRI should be infrastructure — the kind of tool you forget was ever new.

04
Open code is better code.

Not for moral reasons (though those too). For practical ones — bugs are found faster, security is verified, longevity is guaranteed, and the community that forms around shared tools always surprises you.

/// supporting the project

Sustain it without selling out.

Open source sustainability is hard. Here's how we'll try.

  • GitHub Sponsors. Recurring support for the maintainers. Voluntary. [coming soon]
  • Open Collective. Transparent finances. All sponsorships go through the foundation. [coming soon]
  • Hosting tiers (eventually). Managed MRI instances for teams. Optional, transparent pricing, core stays free.
  • Support contracts (eventually). For teams that need guaranteed response times. Revenue funds open-source work.
  • No VC. We won't take venture money. The pressure to "grow fast and monetize" is incompatible with the manifesto.
